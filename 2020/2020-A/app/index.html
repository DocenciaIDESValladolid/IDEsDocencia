Descripción del proyecto

Una app de gran ayuda para planificar tu propio Camino de Santiago y no perderte ningún detalle de esta experiencia.

Los datos geoespaciales están cobrando gran relevancia en todos los sectores.

Gracias a esta app, podrás planificar tu ruta día a día, además de añadir paradas y visitas a lugares de interés a lo largo del Camino de Santiago.

  1. Uso de tecnologías IDE - La aplicación utiliza datos y servicios distribuidos enlazados mediante recomendaciones OGC.
  2. Base de datos espaciales  - Gestiona la información en una base de datos PostgreSQL/PostGIS.
  3. Nodo IDE - Servidor Geoserver configurado como nodo IDE con la información propia de la aplicación.
  4. Applicación móvil - Aplicación de cliente ligero con Javascript para implementar el prototipo de un servicio geo-referenciado.

Instrucciones

Estos son los pasos que debes seguir para sacar el mayor rendimiento de la aplicación.

  1. Selecciona tus contenidos favoritos - Aeropuertos, Estaciones de tren, Áreas de servicio... ¡y muchas más! Clica sobre el icono de información de la parte inferior de la pantalla y selecciona qué información deseas que se muestre sobre el mapa.
  2. Escoge tu próximo destino  - Clica sobre el destino en el mapa o introduce la distancia que deseas caminar en el desplegable de la izquierda.
  3. Cambia los datos que quieres ver - ¿Demasiada información? Haz click sobre el menú de capas en la parte inferior de la pantalla y oculta tantas capas como quieras.
  4. Vive la experiencia - Disfruta del Camino de Santiago y de todo lo que lo rodea. Del resto, nos encargamos nosotros.

Technology Stack

En esta aplicación, se han usado las siguientes tecnologías

Define el esqueleto de la página web, el cual luego es accedido y modificado por JavaScript.

HTML
Lado del cliente

Lleva a cabo la interacción con el usuario, así como la lógica que debe seguir la aplicación. Se encarga también de las consultas externas, la adecuación de los datos.

JavaScript
Lado del cliente

Framework que simplifica la interacción entre JavaScript y el código HTML, así como el manejo de eventos.

JQuery
Lado del cliente

Se encarga de las tareas de mostrar y dibujar los componentes geográficos (mapa, marcadores, ruta, etc).

OpenLayers
Lado del cliente

Además de hacer de servidor web, se encarga de ofrecer servicios OGC (WMS, WFS y WPS).

GeoServer
Lado del servidor

Base de datos relacional que sirve la capa local a GeoServer para que este ofrezca sus servicios.

PostgreSQL
Lado del servidor

Servidor HTTP Java en el que se aloja la aplicación y geoserver.

JETTY
Lado del servidor

Servidores externos cuya labor será proveer a nuestra aplicación de los datos necesarios.

Servidores IDEE
Lado del servidor

Conclusiones y líneas de trabajo futuras

Este proyecto recoge el uso de todas las tecnologías vistas en la asignatura de IDEs.

  1. Realización de operaciones espaciales - Calcula la ruta que debe seguir el usuario y muestra los puntos de interés que ha seleccionado.
  2. Emplea servicios OGC  - Hace uso de servidores externos gestionados por la entidad competente para obtener información relevante para el usuario.

No obstante, aún quedan líneas de mejora interesantes que podrían aportar mucho valor a la aplicación

  1. Incluir más puntos de interés 
  2. Ofrecer valoraciones de los sitios de interés 
  3. Añadir más variantes del Camino de Santiago 
  4. Compartir la ruta en redes sociales 
  5. Seguimiento de la actividad en tiempo real 
  6. Seguimiento diario de la actividad 
  7. Chat con usuarios de la aplicación cercanos a tu ubicación
